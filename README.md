# FTH 订单自动化助手 (FTH Order Automation Assistant) v3.0

一个基于 React + Vite 开发的高效订单处理工具，专为物流订单自动化设计。支持福鹿家系列订单处理及商颂下单转换，具备智能查重、规则匹配及一键导出功能。

![Version](https://img.shields.io/badge/version-3.0-blue)
![License](https://img.shields.io/badge/license-MIT-green)

## 核心特性

- **📦 福鹿家自动化工作流**：
  - **第一步：提取门店信息** - 自动过滤历史重复订单，精准提取合格门店。
  - **第二步：生成 ERP 订单** - 自动关联货品模板，批量生成符合吉客云导入标准的 Excel。
  - **第三步：回填快递单号** - 智能匹配吉客云与菜单屏物流单号，实现 OMS 单号批量回填。
- **🚀 商颂下单转换**：
  - 支持复杂界面的订单文本解析。
  - 智能补纸逻辑：根据打印机类型自动匹配对应的纸张规格。
  - 深度物流匹配：内置快递价格明细规则，自动根据省份、平台及重量计算最佳物流。
- **🎨 极致交互体验**：
  - **现代审美**：采用 Apple 风格设计语言，支持深色/浅色模式无缝切换。
  - **实时反馈**：内置模拟终端记录执行日志，关键错误实时弹窗提醒。
  - **表格预览**：在导出前支持实时数据预览及红色标注查重项。

## 技术栈

- **前端框架**: React (TypeScript)
- **构建工具**: Vite
- **UI 组件**: Tailwind CSS + Lucide Icons
- **数据处理**: XLSX (xlsx-js-style)
- **状态提示**: Sonner

## 更新日志 (2026-04-22)

### 1. 全系统“防呆”校验逻辑升级
- **全槽位身份识别**：实现了基于文件内容（表头特征、关键字、列组合）的智能识别系统。
- **交叉误操作拦截**：严禁在错误位置上传文件（例如：在快递单槽位上传采购订单）。系统会自动检测文件身份并给出精准的重定向提示。
- **模块物理隔离**：强化了“福鹿家”与“商颂”业务文件的互斥校验，防止品牌间数据混淆。
- **错误指引标准化**：统一所有报错文案为“请上传正确的：[具体文件名]”，并明确告知检测到的冲突类型。

### 2. 核心功能修复与优化
- **第 2 步手动替换修复**：解决了手动上传文件失败后静默回退到旧内存数据的 BUG。现在手动上传具有最高优先级，且解析失败会强制报错停止，确保数据准确性。
- **第 3 步精准匹配增强**：
    - 解决了“回填目标表”与“匹配数据源”因文件名相近导致的传反问题。
    - 引入了基于 `快递公司` 与 `收货人` 核心字段的唯一性锁定。
- **物流匹配逻辑升级 (App.tsx)**：
    - **正则增强**：重写了识别算法，支持所有主流快递前缀（SF, KY, DPK, JT 等）后紧跟单号的格式。
    - **支持分列格式**：增加了对 Excel 双列格式的支持（A列姓名、B列单号），不再强制要求姓名与单号合并在同一格。
    - **双库交叉搜索**：实现了逻辑回退机制，如果当前订单在对应库中未找到，会自动尝试在备用库中匹配，提升成功率。

### 3. UI/UX 体验改进
- **执行终端日志增强**：所有文件上传操作现在都会在终端打印详细的表头检测信息，极大地方便了排错和调试。
- **校验反馈即时化**：所有校验在文件选择的一瞬间完成，无需等待点击执行，防错响应更迅速。
